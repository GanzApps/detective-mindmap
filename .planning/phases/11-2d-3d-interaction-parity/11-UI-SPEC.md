### Focus / Dim Opacity Contract

| State | 2D opacity | 3D opacity |
|-------|-----------|-----------|
| Active (in focused neighborhood) | 1.0 | 1.0 |

## Interaction Contract

Phase 11 is primarily an interaction phase. These contracts govern behavior, not just appearance.

### 3D Node Drag (INTER-07)

| State | Visual Signal |
|-------|--------------|
| Node not selected | No drag affordance. Cursor remains default. Canvas drag = camera pan/rotate. |
| Node selected (click) | Node renders with accent ring (#7c3aed, 2px screen-space). This is the only drag-eligible state. |
| Drag active on selected node | Node moves in screen plane. Ring stays visible during move. No ghost/preview needed. |
| Drag released | Node stays at new position (screen-plane mapped to 3D world position). Ring remains until deselected. |

Disambiguation rule: If pointer-down lands on a selected node (within hit-test radius), motion = node move. Otherwise motion = camera rotate/pan. No modifier key required. Source: CONTEXT.md locked decision.

### 2D Node Placement Persistence (INTER-08)

| Event | Behavior |
|-------|---------|
| Drag end in 2D | `fx` and `fy` are set and kept on the simulation node datum. Node is pinned for session lifetime. |
| Graph data reload | `fx`/`fy` reset (positions are session-only). |
| Page reload | Positions reset. |

### Selection / Focus State Parity (INTER-06)

| Event | 2D | 3D |
|-------|----|----|
| Node click | `selectedNodeId` updated | `selectedNodeId` updated (same prop) |
| Focus neighborhood | `highlightedNodeIds` applied | `highlightedNodeIds` applied (same prop) |
| View switch 2D → 3D | `selectedNodeId` preserved (no clear on `viewMode` change) | Receives same `selectedNodeId` |
| View switch 3D → 2D | Same — no reset | — |

Source: CONTEXT.md — "Focus state survives view switch." Plumbing already exists via `GraphWorkspace` props.

### Cross-Renderer Position Carry-Over (INTER-08)

| Direction | Mapping |
|-----------|---------|
| 2D → 3D | 2D node `{x, y}` → 3D `{x: node.x, y: 0, z: node.y}` (best-effort, screen-plane) |
| 3D → 2D | 3D node `{x, z}` → 2D `{x: node.x, y: node.z}` |

Stored in a shared `nodePositionMap: Record<string, {x: number, y: number}>` at `GraphWorkspace` level. Updated on drag-end in either renderer.

### 2D Icon Asset Rendering

| State | Behavior |
|-------|---------|
| Icon available for entity type | Render icon image centered inside node shape, at 60% of node radius |
| Icon not available (fallback) | Use Phase 10 canvas-drawn shape with entity color fill |
| Icon loading (async) | Render Phase 10 fallback until asset is decoded |

Icon assets: SVG files loaded via `new Image()` and cached in a module-level `Map<EntityType, HTMLImageElement>`. One image per entity type.
