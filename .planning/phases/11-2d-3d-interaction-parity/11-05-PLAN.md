# Phase 11-05 Plan: Drag Parity Fixes

## Goal

Close the remaining Phase 11 UAT failures around drag behavior so shared placement parity actually works in live use, not just in tests.

## Gaps Being Closed

### Gap R1

Truth:
- Click and hold any node in 2D, drag - node moves with cursor and pins at new position on release.

Observed:
- `2D` node drag still does not move at all in live use.
- Linked child nodes that should remain emphasized are dimming incorrectly in `2D`.

### Gap R2

Truth:
- Drag upward in 3D moves node up. After drag plus view switch, node appears in same visual quadrant in 2D.

Observed:
- `3D` drag can still feel inverted or unclean after camera rotation.
- Shared carry-over between `3D` and `2D` still does not feel visually trustworthy.

## Work Plan

### Task 1: Fix live 2D drag activation path

Files:
- `src/components/graph/ForceGraph2D.tsx`

Work:

- correct the drag subject / pointer source so real node drags start reliably
- verify that right-pan support does not block left-node drag
- keep dragged nodes pinned after release

Done when:
- left-drag on a node moves it reliably in 2D

### Task 2: Fix 2D focused-neighborhood dimming logic
