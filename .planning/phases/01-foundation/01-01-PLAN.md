**jest.config.ts (per D-04):**

Create `jest.config.ts` at the project root with ts-jest preset and node test environment:

```typescript
import type { Config } from 'jest';

const config: Config = {
  preset: 'ts-jest',
  testEnvironment: 'node',
  moduleNameMapper: {
    '^@/(.*)$': '<rootDir>/src/$1',
  },
  testMatch: [
    '**/__tests__/**/*.test.ts',
    '**/*.test.ts',
  ],
  transform: {
    '^.+\\.tsx?$': ['ts-jest', {
      tsconfig: {
        strict: true,
      },
    }],
  },
};

export default config;
```

This ensures:
- TypeScript test files are processed by ts-jest (not Babel)
- Tests run in a Node.js environment (no jsdom — lib/graph/ has no DOM dependency per D-26)
- The @/* path alias resolves correctly in test files
